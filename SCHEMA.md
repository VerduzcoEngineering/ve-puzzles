# Puzzle card schema

Each `puzzles/VE-PUZ-XXXX.json` is one card. `index.json` is the array of all cards,
regenerated by `scripts/build-index.mjs`.

```jsonc
{
  "id": "VE-PUZ-0001",          // string, stable, unique. Cited in URLs + posts.
  "kind": "code",               // "code" | "schematic" | "spec"
  "lang": "c",                  // optional — snippet language for syntax highlight
  "topic": ["embedded"],        // string[] — tags + social hashtags
  "difficulty": "intro",        // "intro" | "journeyman" | "gnarly"
  "prompt": "…",                // the scenario / question above the snippet
  "snippet": "…\n…",            // code or ASCII schematic, rendered monospace
  "choices": ["…", "…"],        // multiple-choice options
  "answerIndex": 1,             // 0-based index into choices of the correct answer
  "reveal": "…",                // the diagnosis, shown after answering
  "lesson": "…",                // the engineering-judgment takeaway
  "hasBreakdown": false,        // true if a matching VE-PUZ-XXXX.md breakdown exists
  "caption": "…",               // optional — hook line for a social post
  "hashtags": ["embedded"]      // optional — social-only
}
```

## Rich breakdown (optional)

For sophisticated puzzles that need equations or diagrams, add a sibling markdown
file `puzzles/VE-PUZ-XXXX.md` (pure markdown body, no frontmatter) and set
`hasBreakdown: true`. The website renders it at `/puzzles/<id>` with KaTeX for math
and inline SVG/images for figures. Keep figures as SVG where possible — they stay
crisp and theme with the site.
